# Главное здание

## Описание
Добавить главное здание (HQ), которое принимает обработанные ресурсы и отображает их количество в верхней части экрана. Ресурсы отображаются без названий, только цифры. При количестве более 1000 используется приставка "к" (например, 3132 → 3к). Дробные части не отображаются. Главное здание не принимает необработанные ресурсы (руды).

## Оценка сложности
**Средняя (6/10)**

- Новый entity_type (главное здание)
- Новая таблица для хранения ресурсов главного здания (user_resources)
- UI компонент для отображения ресурсов в header
- Логика приёма ресурсов от транспортных систем
- Фильтрация необработанных ресурсов

## Оценка интересности
**Высокая (9/10)**

Это ключевая механика для прогресса игры. Даёт игроку чёткую цель (накопление ресурсов) и визуальный фидбек о достижениях.

## Краткий план реализации

1. **База данных**:
   - Создать таблицу `user_resource` с полями: user_id, resource_id, quantity
   - Добавить entity_type для главного здания

2. **Backend**:
   - ResourceTransport: проверка, является ли целевая entity главным зданием
   - При доставке ресурса в HQ: добавлять в user_resource вместо entity_resource
   - API для получения user_resource

3. **Frontend**:
   - Header компонент для отображения ресурсов
   - Функция форматирования чисел (1000+ → "к")
   - Иконки ресурсов без подписей
   - Обновление при изменениях через ResourceRenderer

4. **Спрайты**:
   - Создать красивый спрайт главного здания (больше других построек)
